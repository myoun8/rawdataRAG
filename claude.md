# rag-knowledge-pack-template

A RAG knowledge-pack template for NIST NCNR neutron-scattering instrument documentation. Four packs: `candor/`, `vsans/`, `nse/` (instrument-specific) and `common/` (shared NICE/NCNR-wide docs).

## Pack structure (per `PACK_STRUCTURE.md`)

Each pack (`<pack>/`) contains:
- `README.md`, `source_inventory.csv`, `manifest.jsonl`, `glossary.yaml`, `access_policy.yaml`, `ingest_config.yaml`
- `originals/` — unmodified source material (web_pages, pdfs, papers, manuals, notebooks, scripts, data_examples)
- `normalized/` — Markdown docs with YAML frontmatter (doc_id, source_id, instrument, workflow_stage, source_type, access_level, status, owner, last_reviewed, source_url_or_path, citation_required), organized into stage subfolders (overview, instrument_control, experiment_planning, data_access/raw_data, metadata, sample_environment, troubleshooting, citations_publications, etc.)
- `chunks/` — JSONL chunk records (chunk_id, doc_id, source_id, text, metadata) generated from `normalized/`
- `eval/` — JSONL evaluation questions for retrieval testing
- `review/` — review artifacts

Schemas for these record types live in `schemas/`.

## Scripts (`scripts/`)

The embedding/vectorstore/chat-model layer is built on LangChain (`langchain-core`, `langchain-ollama`, `langchain-chroma`, `langchain-openai`). Normalization uses the Groq API directly via the `groq` SDK. `GROQ_API_KEY` is read from a `.env` file in the repo root (or the environment).

- `run_pipeline.py` — **main entry point** that chains all four ingestion steps in order: normalize → chunk → validate → embed. Reads `GROQ_API_KEY` from `.env` or the environment; exits early with a clear message if it is missing. Run as `python scripts/run_pipeline.py [--pack PACK] [--model NAME] [--skip-normalize] [--skip-validate] [--dry-run]`. `--model` defaults to `moonshotai/kimi-k2-instruct`; `--pack` scopes normalization/chunking/validation to one pack (embed always ingests all available chunk files).
- `full_document_ingestion.py` — converts files in each pack's `originals/` to normalized Markdown using the Groq API. Skips files that already have a corresponding normalized `.md`. For each new file, streams the model's output then asks the user to confirm the target workflow stage before writing. Run as `python scripts/full_document_ingestion.py --model NAME [--api-key KEY] [--pack PACK] [--dry-run]`. `--api-key` falls back to `GROQ_API_KEY` env var. PDF support requires `pypdf` (`pip install pypdf`).
- `chunk_markdown.py` — splits `normalized/**/*.md` by H2 headings into `<pack>_chunks.generated.jsonl`. Stdlib only. **Not using LangChain by design**: its two-pass algorithm (split H2 → split oversized H2s by H3 → merge small consecutive sections back up to a size target) has no equivalent among LangChain's text splitters — reimplementing on top of them would require writing the same custom merge logic with no benefit. Run as `python scripts/chunk_markdown.py <pack>`.
- `embed_and_ingest.py` — embeds every pack's `chunks/*_chunks.jsonl` with `langchain_ollama.OllamaEmbeddings` (`nomic-embed-text` via Ollama) and loads them into a `langchain_chroma.Chroma` vectorstore (backed by a `chromadb.PersistentClient` at `./chroma_db`, collection `ncnr_rag`). Reaches into the wrapped raw collection (`vectorstore._collection.add(...)`) to embed an enriched string (doc_id/section prefix + `search_document:` instruction prefix) while storing the original chunk text — `Chroma.add_texts()` has no precomputed-embeddings parameter. Also writes `chunk_id` into metadata so eval scripts can scope results back to a pack's own chunks (LangChain `Document` objects from similarity search don't reliably expose the Chroma-assigned id). Requires Ollama running with `nomic-embed-text` pulled (`ollama pull nomic-embed-text`).
- `validate_pack.py` — validates a pack's required files/dirs, JSONL syntax, chunk/metadata field completeness, and cross-references chunk `source_id`s against `source_inventory.csv`. Exits non-zero on any error, which causes `run_pipeline.py` to abort. Run as `python scripts/validate_pack.py <pack>`.
- `query_rag.py` — end-to-end RAG query against an LLM. Retrieves from the `langchain_chroma.Chroma` `ncnr_rag` collection via `similarity_search_with_score()` (filtered by `status=current`, `access_level` cascade, optional `instrument`/pack; score is cosine distance, used by `--max-distance`), then runs an LCEL chain (`ChatPromptTemplate | llm | StrOutputParser()`) with the retrieved chunks as context. Three backends: `--backend ollama` (default) uses `langchain_ollama.ChatOllama`; `--backend openai` uses `langchain_openai.ChatOpenAI` against any OpenAI-compatible endpoint (e.g. NIM via SSH tunnel) — requires `--base-url` and `--model`; `--backend ssh` uses a custom `SSHChatModel` to curl a NIM container's loopback over SSH without a local tunnel — requires `--model`, `--ssh-host`, `--ssh-user`, and `--ssh-password` or `--ssh-key-file` (or `GB10_SSH_*` env vars). `--max-distance D` (default `0.5`, range 0–2) skips the LLM call if the closest chunk exceeds D. Requires `embed_and_ingest.py` to have been run first and Ollama running with `nomic-embed-text` pulled. Run as `python scripts/query_rag.py "<question>" [--pack PACK] [--top N] [--backend ollama|openai|ssh] [--base-url URL] [--model NAME] [--api-key KEY] [--ssh-host HOST] [--ssh-user USER] [--max-distance D] [--access-level public|internal|restricted]`.
- `test_retrieval_embedding.py` — embedding-based retrieval evaluation: runs each pack's `eval/*.jsonl` questions against the `ncnr_rag` Chroma collection and reports top-1/top-k accuracy and MRR. Pack-scoped chunk filtering uses the `chunk_id` metadata field written by `embed_and_ingest.py`. Requires Ollama with `nomic-embed-text` pulled. Run as `python scripts/test_retrieval_embedding.py <pack> --evaluate [--detail]` or `--evaluate-all [--output-csv file.csv]`.
- `evaluate_retrieval_ragas.py` — embedding-based retrieval evaluation using RAGAS-standard Context Precision@K and Context Recall, computed directly from each eval question's `expected_sources` ground truth. Retrieves the same way as `test_retrieval_embedding.py`. Run as `python scripts/evaluate_retrieval_ragas.py <pack> [--top N] [--detail]` or `--evaluate-all [--output-csv file.csv]`.

`requirements.txt` pins `chromadb`, `paramiko`, `groq`, `pypdf`, and the LangChain integration packages (`langchain-core`, `langchain-ollama`, `langchain-chroma`, `langchain-openai` — the slim per-integration packages, not the umbrella `langchain` package). None pull in `sentence_transformers`/PyTorch. All embedding-dependent scripts require Ollama running locally with `nomic-embed-text` pulled (`ollama pull nomic-embed-text`); `query_rag.py`'s `--backend ollama` additionally needs a chat model pulled (e.g. `ollama pull llama3.2`).
